[机器学习基础：预测泰坦尼克号乘客生还率](https://trial.learnincode.com/hub/user-redirect/git-pull?repo=http%3A%2F%2F172.17.0.8%2Flearnincode%2Ftitanic_survive&urlpath=tree%2Ftitanic_survive%2F%E6%9C%BA%E5%99%A8%E5%AD%A6%E4%B9%A0%E5%9F%BA%E7%A1%80-%E9%A2%84%E6%B5%8B%E6%B3%B0%E5%9D%A6%E5%B0%BC%E5%85%8B%E5%8F%B7%E4%B9%98%E5%AE%A2%E7%94%9F%E8%BF%98%E7%8E%87.ipynb)

主要特点：
1. 线上编程，随时随地打开网页就能编写和运行代码。
2. 学习教程和可编写运行的代码框在一个网页中为你呈现，免去你来回切换视频、文档、编程环境之苦。
3. 录屏演讲：老师的每一次鼠标移动、键盘上按下的每个字母都在网页上为你重现，效果就像老师在你身边手把手指导。 这种方式能降低对老师的要求。
4. 运行环境是可以定制的容器服务，可根据需要预装第三方库，也可以在网页中安装第三方库。
5. 这里的录屏，其实是录“操作”。这种技术，比视频更加直接，用户无需在视频和Notebook之间切换。录制的操作，你能看清代码。如果是视频的话，受限于网速，可能会出现代码看不清，不同操作系统或浏览器之间代码的字体/格式有所差异等问题。


## 远程异步录屏答疑

[模拟学生练习答疑]()

主要特点：

1. 老师可以直接访问学生的notebook及其专属的运行环境。针对学生练习中存在的问题，老师可以录制一份答疑演讲，为他解答。效果就跟一对一答疑一样。同时这中答疑又是异步的，也就是说，老师和学生不需要直播面对面沟通答疑，而是将操作录制下来，待学生下次登录学习时就能观看解答。
2. 有了这个功能，上课流程可以变成：学员观看录屏讲解 -> 完成练习 -> 老师批改（添加录屏讲解）-> 学员观看老师批改的讲解，理解错题 -> 老师电话回访，是否还有不理解之处，通过一轮录屏批改后仍然不理解的情况下，再需要直播方式的解答。
3. 通过这种方式，能大大提升老师的教学人效，降低师资要求。
